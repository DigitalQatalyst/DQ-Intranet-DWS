# GHC Service Detail Page Bugfix Design

## Overview

The GHC service detail page currently displays identical static content for all 8 GHC service cards instead of showing their specific content from the `GUIDE_CONTENT` constant. This design addresses the bug by implementing dynamic content loading based on the service ID, ensuring each GHC element displays its unique content, proper tab structure, and correct metadata.

The fix involves modifying the `ServiceDetailPage.tsx` component to detect GHC services and render them using the rich content available in `src/constants/guideContent.ts`, while preserving the existing behavior for non-GHC services.

## Glossary

- **Bug_Condition (C)**: The condition that triggers the bug - when a GHC service ID is accessed but displays generic static content instead of specific GHC content
- **Property (P)**: The desired behavior when GHC services are accessed - display specific content from `GUIDE_CONTENT` with proper tabs and metadata
- **Preservation**: Existing non-GHC service behavior and page structure that must remain unchanged by the fix
- **ServiceDetailPage**: The React component in `src/pages/marketplace/ServiceDetailPage.tsx` that renders service detail pages
- **GUIDE_CONTENT**: The constant in `src/constants/guideContent.ts` containing rich content for each GHC element
- **GHC Service IDs**: The 8 service identifiers: 'ghc', 'dq-vision', 'dq-hov', 'dq-persona', 'dq-agile-tms', 'dq-agile-sos', 'dq-agile-flows', 'dq-agile-6xd'

## Bug Details

### Bug Condition

The bug manifests when a user navigates to any GHC service detail page. The `ServiceDetailPage` component fetches data from the Supabase `services` table, but for GHC services, this returns generic static content instead of the rich, specific content that exists in the `GUIDE_CONTENT` constant.

**Formal Specification:**
```
FUNCTION isBugCondition(input)
  INPUT: input of type { serviceId: string, serviceData: ServiceDetail }
  OUTPUT: boolean
  
  RETURN input.serviceId IN ['ghc', 'dq-vision', 'dq-hov', 'dq-persona', 'dq-agile-tms', 'dq-agile-sos', 'dq-agile-flows', 'dq-agile-6xd']
         AND input.serviceData.title == "DQ Associate Owned Asset Guidelines"
         AND NOT displayingSpecificGHCContent(input.serviceId)\nEND FUNCTION\n```\n\n### Examples\n\n- **Vision Service**: User clicks on Vision card, expects to see \"The Vision (Purpose)\" with purpose-driven content, but sees \"DQ Associate Owned Asset Guidelines\" instead\n- **Agile TMS Service**: User clicks on Agile TMS card, expects to see \"Agile TMS (Tasks)\" with task management content, but sees identical generic guidelines\n- **HoV Service**: User clicks on HoV card, expects to see \"The HoV (Culture)\" with 4 tabs (Overview, Understand, Learn & Practice, Other Materials), but sees only 2 tabs (Overview, Other Materials)\n- **Tab Structure**: All GHC services should show 4 tabs but currently show only 2 generic tabs\n\n## Expected Behavior\n\n### Preservation Requirements\n\n**Unchanged Behaviors:**\n- Non-GHC services (like actual \"DQ Associate Owned Asset Guidelines\") must continue to display current static content and structure\n- Page layout, styling, responsive design, and navigation must remain unchanged\n- Breadcrumb navigation and \"More Detail\" button functionality must continue to work\n- Service summary sidebar structure and \"Request Service\" / \"Save for Later\" buttons must remain functional\n- Footer and header components must continue to work as expected\n\n**Scope:**\nAll inputs that do NOT involve GHC service IDs should be completely unaffected by this fix.
